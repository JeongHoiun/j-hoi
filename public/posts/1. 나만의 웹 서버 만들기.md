개인적으로 가지고 놀 웹 서버를 만들고 싶어져서 시작하게 되었다.
배우고 싶은 것들을 모두 집어 넣을 예정.
이 서버에서 개인 블로그도 운영하고 나의 커리어를 공개하여 영향력을 높이고 싶었다.

나만의 웹 서버를 만들기 위해서 고려중인 것들을 이 포스트에 적어보려고 한다.
## Next.js
가장 먼저 내가 배워보고 싶은 기술이다. 이미 이전에 몇 번 만져보긴 했지만, 깊은 이해를 가져가지 않기도 했고, 13.4.0 버전 이후로 안정화된 App Routing 방식을 사용해보고 싶었다. React에 대한 지식도 예전에 머물러 있는 느낌이라서 React Server Component 라던지, 최근 여러 곳에서 핫하게 포스팅되고있는 React Compiler에 대해서도 알아보기 위해 일단 최신버전의 Next.js, React를 사용하여 개발을 해 보려고 한다. (버전 업그레이드는 나중에 고려...)
## AWS
재작년에 회사에서 처음으로 Cloud 서비스를 개발해본다고 하면서 Naver Cloud를 이용했었는데 그때 나는 Frontend 개발에 집중하고 있어서 IaaS에 대해서는 제대로 공부해보지 못했다. 시작은 단순히 EC2에 서버를 올리는 것으로 시작하게 되겠지만 추후 S3에 연결하거나 지속적 배포 등을 위해 서비스를 이용해 볼 수 있을 것 같다.
## Self-hosting
나만의 웹 서버를 가지기 위해서는 나만의 도메인이 필요하다고 생각했다. 대학교 때 스스로 도메인을 구입하여 서버를 가지고 있던 친구가 있었는게 그게 멋있어 보였다. 너무 많은 시간이 지났지만 이제 경제적으로도 여유로운 이 시점에 나만의 도메인, 나만의 웹 서버가 있으면 재밌을 것 같다.
## 기술 블로그
웹 서버만 있으면 의미가 없다. 기술 블로그를 만들어서 내 지식을 저장해 둘 공간이 필요하다. 필요에 따라서 지식을 공유하고 사람들과 소통할 수도 있을 것 같다.
## SEO
Cloud 서비스 개발을 할 때 가장 중요하게 생각하는 요소였는데 회사에서는 해당 클라우드 서비스를 B2C 서비스로 생각하지 않았던 것 같다. 그저 영업을 통한 홍보를 하려고 한 것 같은데 나는 웹 서버를 호스팅할 때는 SEO가 가장 중요하다고 생각한다. 초반에는 SEO를 고려한 개발이 없을 수 있으나 어느 정도 개발이 되면 검색엔진에 내 웹 서버가 나올 수 있도록 고려가 필요할 것 같다.
